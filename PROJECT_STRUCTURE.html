🔐 Authentifizierungs-Flow

1. Registrierung:
Frontend: POST /api/auth/signup → Backend: Password hashen (BCrypt)
→ User + Role speichern → 200 Success

2. Login:
Frontend: POST /api/auth/signin → Backend: Credentials verifizieren
→ JWT generieren (HMAC-SHA256, 24h) → Return Token + User Data

3. Authenticated Requests:
Frontend: localStorage.getItem('user.token')
→ Authorization Header: "Bearer <token>"
→ JwtAuthFilter validiert Token
→ SecurityContext setzt User + Roles
→ Controller prüft @PreAuthorize(hasRole('ADMIN'))

4. Token Struktur:
Header: HMAC-SHA256
Payload: { sub: "username", iat: 1234567890, exp: 1234567890, roles: "ROLE_USER" }
Signature: Secret Key (app.jwt.secret)
